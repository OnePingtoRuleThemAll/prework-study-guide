Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible contenet shown to the user.
  • HTML: Hypertext Markup Language
  • HTML is made up of elements which work together to provide content and the basic framework for all pages on the web, this is your foundation.
  • Elements are a type of content on a webpage. They can be text, images, headers and lists. Also used to divide content on a page.
  • HTML attributs are ALWAYS defined in the beginning element, never the closing element
  • Metadata: Data about a webpage that the browser uses to effectively serve a webpage to users, but it is not usually seen by users themselves. It contains info about accessibility, search engines and performance.
  • id attribute: used in HTML as a way to identify elements and select them.

CSS

  • CSS: Cascading Style Sheets
  • 3 ways to style: Inline CSS, Internal CSS style sheet, External CSS style sheet.
  • Inline: Used to style and element directly in the HTML file. Useful for quick and permanent changes but are less flexible.
  • Internal SS: Contains CSS rules for the webpage in the head section of the HTML file. The rules only apply to that page.
    • External SS: Most common way to style a webpage. Is preferred in most cases as it allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • CSS= STYLE
  • Common element types are: meta-related, content division, and semantic.
  • charset: character set
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • src attribute: defines the location or source of the image file.
  • alt attribute: contains a text string that describes the appearance and functionability of the image.
  • section elements are semantic elements that define a section in a document that contains thematic content
  • Separation of concerns: refers to separating code into different files based on use. Based on the idea that each section of code should have its own responsibility.
  • Declarations: contain two components: the CSS property we want to apply and the value of the property. (EX: the declaration color:blue; includes the CSS property "color" and the value for that property "blue")
  • DRY: Do Not Repeat Yourself: a coding princliple that encourages developers to reuse or share pieces of code in order to reduce the number of line of code that need to be written or maintained.

ON THE JOB

  • ORDER MATTERS: When using headings, it's important to use heading tags in order from h1 to h6 to make the page accessible to users who access the page using screen readers or other devices. The biggest heading (h1) should only appear once on each page
  • The alt text is important for screen readers and other assistive technologies. Alt text can also improve search engine rankings for your website
  • Developers test code early and often to catch bugs. Sometimes testing can be as simple as looking for changes in the browser to seee if the webpage is rendering as expected.
  • It is important to frequently sync the local version of your code with the remote version in GitHub. (This matters especially when working on a team, so that functional, updated version of the project can be shared and accessed by all team members.)
  • Working on an outdated version can lead to big sync problems
  • In most real-world jobs, when creating a pull request, you would assign other engineers to review your code before merging.
