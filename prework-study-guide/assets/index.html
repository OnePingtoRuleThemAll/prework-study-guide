HTML

  • The head element contains information about the webpage.
  • The body element represents the visible contenet shown to the user.
  • HTML: Hypertext Markup Language
  • HTML is made up of elements which work together to provide content and the basic framework for all pages on the web, this is your foundation.
  • Elements are a type of content on a webpage. They can be text, images, headers and lists. Also used to divide content on a page.
  • HTML attributs are ALWAYS defined in the beginning element, never the closing element
  • Metadata: Data about a webpage that the browser uses to effectively serve a webpage to users, but it is not usually seen by users themselves. It contains info about accessibility, search engines and performance.
  • id attribute: used in HTML as a way to identify elements and select them.

CSS

  • CSS: Cascading Style Sheets
  • 3 ways to style: Inline CSS, Internal CSS style sheet, External CSS style sheet.
  • Inline: Used to style and element directly in the HTML file. Useful for quick and permanent changes but are less flexible.
  • Internal SS: Contains CSS rules for the webpage in the head section of the HTML file. The rules only apply to that page.
    • External SS: Most common way to style a webpage. Is preferred in most cases as it allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • CSS= STYLE
  • Common element types are: meta-related, content division, and semantic.
  • charset: character set
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • src attribute: defines the location or source of the image file.
  • alt attribute: contains a text string that describes the appearance and functionability of the image.
  • section elements are semantic elements that define a section in a document that contains thematic content
  • Separation of concerns: refers to separating code into different files based on use. Based on the idea that each section of code should have its own responsibility.
  • Declarations: contain two components: the CSS property we want to apply and the value of the property. (EX: the declaration color:blue; includes the CSS property "color" and the value for that property "blue")
  • DRY: Do Not Repeat Yourself: a coding princliple that encourages developers to reuse or share pieces of code in order to reduce the number of line of code that need to be written or maintained.

JavaScript

  • JavaScript= INTERACTION
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in script.
  • Every time you do something with a webpage; clicking a button, retrieving data, or filling out a form, JavaScript is at work.
  • JavaScript features: variables, strings, data types.
  • Invented by Brendan Eich
  • Variables: containers that store values. You start by declaring a variable with the 'let' keyword, followed by the name you give to the variable: let myVariable;
  • A semicolon at the end of a line indicates where a statement ends
  • JavaScript IS case sensitive
  • Variables are necessary to do anything interesing in programming
  • Comments are snippets of text that can be added along with code
  • Operators: Mathematical symbols that produce a result based on two values (or variables)
  • An immutable value is one whose content cannoot be changed without creating an entirely new value
  • Primitive values are immutable-once a primitave value is created, it cannot be changed, althought the variable that holds it may be reassigned another value.
  • Objecs and arrays are mutable by default
  • Boolean type: represents a logical entity and is inhabited by two values: true and false
  • BigInt type: a numeric primitive in JS that can represent integers with arbitrary magnitude.
  • With BigInts, you can safely story and operate on large integers even beyond hte same integer limit for Numbers
  • String type represents textual data and is encoded as a sequence of 16-bit unsigned integer values representing UTF-16 code units
  • Beware of "stringly-typing" our code!
  • Control Flow: the order in which a computer executes code in a script.
  • Computer reads code in a linear fashion, top to bottom
  • conditional statements and loops can be used to control the order in which code is executed
  • Truthy and Falsy: A value is considered 'falsy' if it is one of the following values:0, -0, On, "", null, undefined, NaN or a Boolean datat type with the value of 'false'. All other values, including Boolean with the value of 'true' are considered 'truthy'
  • Array: single variable that is used to hold a group of data. Typically used to hold data that is related in some way
  • Array's are zero-indexed and sequential, meaning that the first item in any array has an index of 0, not 1
  • 'for' loop uses the predicatable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • A 'for' loop contains the following important statements: the starting point, the condition and the final statement which allows the array to iterate over each item
  • Commenting out code: makes it inactive without deleting it, just in case you need it later
  • Function: a set of instructions that tells the computer how to perform a certain task
  • Code Block=Function Body

ON THE JOB

  • ORDER MATTERS: When using headings, it's important to use heading tags in order from h1 to h6 to make the page accessible to users who access the page using screen readers or other devices. The biggest heading (h1) should only appear once on each page
  • The alt text is important for screen readers and other assistive technologies. Alt text can also improve search engine rankings for your website
  • Developers test code early and often to catch bugs. Sometimes testing can be as simple as looking for changes in the browser to seee if the webpage is rendering as expected.
  • It is important to frequently sync the local version of your code with the remote version in GitHub. (This matters especially when working on a team, so that functional, updated version of the project can be shared and accessed by all team members.)
  • Working on an outdated version can lead to big sync problems
  • In most real-world jobs, when creating a pull request, you would assign other engineers to review your code before merging.
  • The console.log method was created to make it easier for developers to test their code. Developers use it daily to debug and test the code that they write.
